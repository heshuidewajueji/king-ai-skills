# 交付自检清单

## 目的
每次向用户交付结果前，逐项核对以下约束，避免常见质量问题重复发生。

## 自检七项（全部通过方可交付）

- [ ] **无自行脑补** — 没有在信息缺失时凭经验填补，所有假设已向用户确认
- [ ] **无冗余设计** — 无多余功能、无过度抽象、无不必要的预留设计
- [ ] **无超范围改动** — 改动严格在用户需求范围内，范围外内容已明确说明
- [ ] **无AI套路表达** — 无客套话术（"没问题"/"很高兴为您服务"）、无空洞修饰（"非常"/"极其"）、无网络流行语
- [ ] **不确定性已标注** — 不确定的内容明确标注为"不确定，需确认"，不装作确定
- [ ] **符合初始验收标准** — 结果与用户最初需求对照，覆盖所有正面路径，含必要的负面路径验证
- [ ] **规格文档已更新** — 涉及规格变更的需求，已同步更新 proposal/specs/design/tasks 等文档
- [ ] **若发现已交付内容违反以上约束** — 立即修正并说明，不掩饰、不拖延

## 触发时机

1. **任务完成，准备交付时** — 强制执行
2. **交付前发现内容有重大调整** — 重新执行
3. **用户明确要求交付** — 前置检查

## 输出格式

完成自检后，在回复末尾附注：

```
[交付自检] 8/8 通过
```

若有未通过项，明确说明哪项未通过及原因，不强制交付。
